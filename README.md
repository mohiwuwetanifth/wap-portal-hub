# wap-portal-hub

## 项目简介

wap-portal-hub 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定的域名或网站，仅作为多个页面资源的集中存放与版本管理用途。

## 目录说明

```
wap-portal-hub/
├── index.html          # 入口页面（可选的导航页）
├── pages/              # 存放各独立 HTML 页面
│   └── ...
├── assets/             # 公用资源（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- **pages/**：放置不同的 HTML 页面文件，每个页面可独立访问。
- **assets/**：存放页面共享的静态资源，便于统一管理和更新。

## 页面归档说明

本仓库中的 HTML 页面均以独立文件形式存在，相互之间没有强依赖关系。每个页面可以单独通过文件路径或部署后的 URL 访问。页面内容可能包含不同的功能或展示用途，具体用途请查看对应页面的注释或内部说明。

## 维护说明

- 新增页面时，请在 `pages/` 目录下创建新的 HTML 文件，并确保所有资源引用路径正确。
- 如需修改公用资源，请更新 `assets/` 下的对应文件，注意保持向后兼容。
- 不建议在页面中硬编码外部域名或敏感信息，以保持页面的通用性和可移植性。
- 提交前请检查页面是否能在无网络环境下正常加载（即不依赖外部在线资源）。

## 部署

本仓库可直接通过 GitHub Pages 或其他静态文件托管服务部署。部署后，所有页面文件将保持原有目录结构，可通过相对路径访问。
